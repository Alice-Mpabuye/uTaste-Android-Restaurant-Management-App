# 1. Retour d’expérience
## 1.1 Organisation de l’équipe
- Communication principale : WhatsApp + rencontre d’équipe à chaque lundi après le cours
- Partage de code : GitHub
- Répartition du travail :
- un responsable de la base de donnée
- un responsable des modèles UML
- un responsable des tests unitaires

On a réparti le travail basé sur les points et on se rencontrait à chaque lundi après le cours pour vérifier notre progrès.

## 1.2 Difficulté rencontrée
### 1. Conflits avec GitHub
- Myriam et Alice travaillaient en même temps, ce qui a causé des problèmes avec GitHub.
- La solution était de manuellement combiner le code (dans un merge sur GitHub) afin que ça fonctionne, ce qui était très compliqué et qui a requis beaucoup de temps pour régler les erreurs causées par le “merge”.
### 2. Ambiguïté avec les exigences
- Certaines exigences étaient ambiguës comme “scanner les ingrédients”. On pensait que les usagers utiliserait des codes QR mais c’était un code bar sur OpenFoodFacts qu’ils utilisent.
- La solution était de demander au client (le prof) pour la clarification et présumé certaine chose nous-même.
### 3. Travaillé ensemble sur une fonctionnalité
- Il était difficile de travailler sur l’ajout d’une fonctionnalité pour l'application avec d’autres personnes, surtout lorsque l’autre personne changeait des composantes principales dans le code.
- La solution était de garder une communication constante avec les membres de l’équipe pour clairement expliquer les composantes ajoutées ou modifiées.

## 1.3 Leçon apprise / Besoins d’un futur projet
### Ce qui s’est bien passé
- Bonne communication entre les membres de l’équipe
- Répartition claire des responsabilités en utilisant Notion
- Ajout des fonctionnalités dans notre application très bien implémentées par chaque membre de l’équipe.
- Penser aux cas exceptionnel après l’ajout d’une fonctionnalité et implémentation de solutions pour chaque possibilité (ex: après l’ajout de la fonctionnalité d'enregistrement de ventes par le vendeur, nous avons pensé à mettre à jour la fonctionnalité existante de réinitialisation de la base de donnée par l’admin pour que ça supprime aussi les ventes).
### Ce qui aurait pu aller mieux
- Dire quand quelqu’un travaille sur quelque chose pour éviter des problèmes avec GitHub.
- Mieux gérer notre temps pour éviter de soumettre les livrables le jour même des dates limites.
